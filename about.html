About Morris

Developer, builder, and careful systems thinker.

I care about software that earns trust: fast pages, clear interfaces, sturdy code, and decisions that are easy to explain later.

My working style

I like to make complexity visible before trying to make it clever. That means mapping user flows, naming tradeoffs, and building small slices that can be tested, reviewed, and improved.

My favorite projects combine product judgment with engineering discipline: dashboards, tools, portfolios, community platforms, and documentation-heavy apps.

Current focus

  • Modern JavaScript and TypeScript applications
  • Accessible, responsive interface systems
  • Fast static websites for GitHub Pages
  • Technical writing and developer education

Values

Clarity

Clear structure beats fragile cleverness. I optimize for future readers, not just the first ship date.

Craft

Interfaces should feel professional in motion, spacing, accessibility, and small edge cases.

Ownership

I prefer to understand the product goal, not just the ticket, so the implementation lands with intent.
